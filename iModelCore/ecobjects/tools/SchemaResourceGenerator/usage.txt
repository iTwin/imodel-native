R"***(
SchemaResourceGenerator inputs [-p 9900-9999] [-c cultureName] [-ns] [-r paths] [-o path] [-of name]

Description:
    Generates resx files with localizable resources from schemas
    and localization supplemental schemas from resx files

Parameter List:
   inputs              Specifies one or multiple input file paths.
   
   -p    9900-9999     Precedence of schema when the input is a resx file
                       Default is 9900.

   -c    cultureName   Only supported when converting from resx to supplemental schema
                       If no culture is provided the culture is determined from the resx filename

   -ns                 No-Supplementation switch. Prevents Schemas from being supplemented.

   -r    paths         Additional paths to look for referenced schemas,
                       separated by semicolons.

   -o    path          Specifies a different output directory.
                       By default, the same directory as the input file is used.

   -of   filename      Use this parameter to specify the base part of the name like:
                       <BaseResxFileName>.ecschema.CultureName.resx
                       By default the output file will be named based on the first input schema.

   /?                  Displays this help message.

Examples:
    SchemaResourceGenerator "c:\test data\House.01.00.02.ecschema.xml" c:\foo\Other.01.00.23.ecschema.xml -of Banana
        Outputs: c:\testdata\Banana.resx
        
    SchemaResourceGenerator c:\SampleSchema.01.00.00.ecschema.xml -ns -r c:\myschema;"c:\other schemas"
        Outputs: SampleSchema.01.00.00.ecschema.resx
        
    SchemaResourceGenerator c:\testdata\House.01.00.00.resx -c en-US
        Outputs: House_Supplemental_Localization_en_US.01.00.00.ecschema.xml

    SchemaResourceGenerator c:\dirOfSchema\ -of SchemaStrings
        Outputs: SchemaStrings.resx
)***"
