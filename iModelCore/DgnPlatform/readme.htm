SDK Delivery

The SDK is delivered in a zip file. When extracting files from the zip file it is important to ensure that the directory structure is preserved. The top level directories and their descriptions are listed below.

  • Demo - Contains source code, sample dgn file for the example programs DgnPlatformExtract, DgnPlatformCreate, LineStyleDemo, and DgnECDemo. DgnPlatformExtract reads information from a design file and saves it to an XML file. DgnPlatfromCreate reads data from an XML file to populate the contents of a DGN file. LineStyleDemo reads linestyles from one resource file and creates them in an output rsc file. DgnECDemo demonstrates some APIs for accessing business property data.
  • Data - Contains seed files, font and linestyle resources.
  • Dll - DgnPlatform dlls
  • Docs - DgnPlatform documentation
  • Include - DgnPlatform include files
  • Libs - DgnPlatform library files

Example Code

Overview

The Visual Studio solution file demo\DgnPlatformDemo\DgnPlatformDemo.sln contains references to four separate Visual C++ projects demo\DgnPlatformDemo\Extract\DgnPlatformExtract.vcproj, demo\DgnPlatformDemo\Create\DgnPlatfromCreate.vcproj demo\DgnPlatformDemo\LineStyle\LineStyleDemo.vcproj, and demo\DgnPlatformDemo\DgnEC\DgnECDemo.vcproj. The extract and create applications work with an XML file that is meant to serve only as a repository of enough DGN data to demonstrate the capabilities of the DgnPlatform SDK. The extract and create example applications are not meant to serve as a translator where round tripped data is identical. The XML format is not documented and will change without warning as additional capabilities are added to the SDK. The LineStyle demo should produce a rsc file with linestyle resources that are visually identical to the input resource file. The DgnECDemo reads some data from demo.dgn and also creates a new file and writes some data.  

Running the examples

There are four batch files (demo\DgnPlatformDemo\runExtractXML.bat, demo\DgnPlatformDemo\runCreateDgn.bat, demo\DgnPlatformDemo\runLineStyleDemo.bat, and demo\DgnPlatformDemo\DgnECDemo.bat) that are set up to work with the delivered example DGN file. The file runExtractXML.bat should be executed before runCreateDgn.bat to create the XML files that will be used by the later.

 

For the exe to run from the command prompt shell, the exe files must be able to locate the appropriate 32 bit or 64 bit dlls. For this to occur the current directory can be set to the location of the dll as is done in the bat files. A second approach would be to set the PATH variable to include the location of the dlls. The exe files may also be copied to the dll directory and run from that location.

Building the examples

The demo\DgnPlatformDemo\DgnPlatformDemo.sln file must be opened from a shell similar to the Visual Studio [x64 Win64 (if 64 bit)]Command Prompt shell where all the variables required by Visual Studio and the C++ compiler and linker are defined. In addition, the example vcproj files require the following environment variables to be set.

DgnPlatformDemo_INCLUDE

DgnPlatform SDK root include directory.

DgnPlatformDemo_LIBS

DgnPlatform SDK root lib directory.

DgnPlatformDemo_TMP

Location for object files and other files created during compiling.

DgnPlatformDemo_OUTPUT

Location for generated executables.

MS_FONTCONFIGFILE

Fully qualified file path of the font configuration file. This can also be controlled by overriding FontAdmin::_GetFontConfigurationFilePath.

MS_FONTPATH

Semi-colon delimited list of directories to search for fonts (all types). This can also be controlled by overriding FontAdmin::_GetRscFontPaths, FontAdmin::_GetShxFontPaths, and FontAdmin::_GetTrueTypeFontPaths.

Two batch files are delivered which set up these variables. The file demo\DgnPlatformDemo\buildDemo.bat can be used to compile the example applications. The file demo\DgnPlatformDemo\openSln.bat can be used to open DgnPlatformDemo.sln in the IDE with the above variables properly set.

DgnPlatformExtract

Overview

This program creates an XML file that reports the Levels, Shared Cell Definitions, Color Table, Models and their graphic elements for the specified DGN file. It will optionally report all glyphs from the default resource-based font as well as the available line-style definitions.

Command Syntax

DgnPlatformExtract dgnfilename [{+-}{GALX}]

where:

+G -G =

enable, disable reporting on glyphs

+A -A =

enable, disable array contents.

+L -L =

enable, disable reporting on line-style resources.

+X -X =

enable, disable XML Fragment reporting.

DEFAULT behavior is: -G +A -L -X
Processing

DgnPlatformExtract reports on the contents of a DGN using the following steps.

  1. Before processing any file data, the "G" command line parameter is tested to see if the glyphs for default resource-based font are to be extracted to the XML file. The default resource font is specified by the ExampleFontAdmin class of the DgnPlatform Host.
  2. Next, the "L" command line parameter is tested to see if line-style definitions are to be extracted to the XML file. If this parameter is set then system (typically RSC based) and current DGN line-styles are processed.
  3. The filename and format is the first file specific data reported. The filename is eventually used to create the name of the DGN file to create when the DgnPlatformCreate application is executed.
  4. Next all dictionary model data is reported. This data is used by all the other models in the file. It includes level definitions, line-style definitions, shared cell definition, and the color table used to define the colors with indices between 0 and 255.
  5. Finally each model, their graphic elements, and reference files are processed. As the graphic elements are processed the appropriate element query interface is used to extract the element data.

DgnPlatformCreate

Overview

This program is used to create a new DGN file and populate it using the data in the XML file that was produced by DgnPlatformExtract.

Command Syntax

DgnPlatformCreate xmlfilename -s"seedfilename" [-o"outputpath"] [-g]

where:

-s"seedfilename"

Seed DGN file. The dimension of the default model of the seed file must match the default model defined in the XML file.

-o"outputpath"

Path of the directory to create the output DGN file in. If this parameter is not specified then the path defined by the environment variable "TMP" is used.

-g

Optional parameter. If specified, this parameter signifies to create a new model for each glyph defined and to create elements that define its strokes.
Processing

DgnPlatformCreate first opens the specified seed file to make sure it is a valid DGN file and then saves it out with the output DGN file name. This output name is generated using the specified output directory and the original DGN file name specified in the XML file. Once the output file is created it is processed using the following steps.

  1. Create data used by all models including color table, levels, and shared cells.
  2. Create model and set it units and global origin to match that specified in the XML file. If the model type is a sheet model then set its SheetDef properties.
  3. Generate elements within the model by looking at the type of data in the XML and determining what element handler to use to create the element.
  4. Once all models have been generated and populated then revisit each model attaching any reference files specified for the model.

LineStyleDemo

Overview

This program is used to read the linestyle data from one linestyle resource file and write the linestyle data to a new resource file.

Command Syntax

LineStyleDemo "inrscfilename" "anydgnfile" "outrscfilename"

where:

inrscfilename

Input resource file containing linestyles.

anydgnfile

Path to a dgn file.

outrscfilename

Output resource file generate from the linestyle data read from the input file.
Processing

LineStyleDemo first opens the specified resource file and DGN file. It then reads the linestyle data for each linestyle in the input resource file and creates a linestyle in the output resource file.

DgnECDemo

Overview

This program demonstrates use of EC APIs to read/write "EC" property data. It should normally be run with the runDgnECDemo.bat to properly setup input parameters.

Command Syntax

DgnECDemo "inputFileName" "seedFileName" "outputDirectory"

where:

inputFileName

Is the fully-qualified filename of demo.dgn.

seedFileName

A seed from which to create the output file.

outputDirectory

Location of the output file, DgnEC.dgn.
Processing

DgnECDemo demonstrates reading legacy ECXAttributes, importing an ECSchema into a file, creating new ECInstances on elements, etc.
